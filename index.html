Overview

This site separates parallel issue tracks so communication can be reviewed chronologically instead of as one merged conflict narrative. It is intended for private sharing, organization, attorney review, or mediation prep.

Important: Review all content before publishing publicly. If the material is sensitive, use a private repository or remove identifying details first.

Publish on GitHub Pages

  1. Create a new GitHub repository.
  2. Upload index.html, styles.css, and README.md.
  3. Open Settings → Pages.
  4. Set Source to Deploy from a branch.
  5. Select the main branch and /root.
  6. Save and wait for the published Pages URL.
